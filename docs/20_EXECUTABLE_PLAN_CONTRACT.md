# Phase 6: Executable Execution Plan Contract

**Date:** 2026-06-05T08:26:09Z  
**Status:** Complete and validated

---

## Overview

Phase 6 upgrades the ExecutionPlan contract from descriptive planning to executable plans.

ExecutionPlanner now generates steps with actionable metadata (selectors, field names, value sources) that future execution layers (ActionExecutor, BrowserAdapter) can use to perform real browser automation.

**Key Improvement:** Plans are now executable, not just descriptive.

---

## Schema Evolution

### Before Phase 6 (Descriptive Only)

```python
@dataclass
class ExecutionPlanStep:
    step_number: int
    action: ExecutionAction
    description: str
    required_fields: list[str]
    optional_fields: list[str]
    estimated_duration_seconds: int
    fallback_action: Optional[ExecutionAction]
    validation_checks: list[str]
```

**Problem:** No execution data
- No selector information
- No value mapping
- No field targeting
- Cannot be executed by browser

### After Phase 6 (Executable)

```python
@dataclass
class ExecutionPlanStep:
    # Original fields (unchanged - backward compatible)
    step_number: int
    action: ExecutionAction
    description: str
    required_fields: list[str]
    optional_fields: list[str]
    estimated_duration_seconds: int
    fallback_action: Optional[ExecutionAction]
    validation_checks: list[str]

    # NEW: Execution metadata (Phase 6 additions)
    selector: Optional[str] = None
    """CSS selector for element interaction"""
    
    field_name: Optional[str] = None
    """Form field name or element identifier"""
    
    value_source: Optional[str] = None
    """Where to get the value: 'user', 'profile', 'static', 'file_path', 'ai_generated'"""
    
    expected_value: Optional[str] = None
    """Expected or static value to use"""
    
    required: bool = False
    """Whether this step is required (can't be skipped)"""
    
    metadata: dict[str, Any] = field(default_factory=dict)
    """Additional execution metadata"""
```

## New Fields Explained

### selector: Optional[str]

**Purpose:** CSS selector for element targeting

**Examples:**
```python
selector="button[type='submit']"
selector="input[name='email']"
selector="textarea[name='headline']"
selector="button.continue-button"
```

**Used by:** BrowserAdapter for finding/clicking elements

### field_name: Optional[str]

**Purpose:** Logical field identifier

**Examples:**
```python
field_name="resume"
field_name="profile"
field_name="email"
field_name="submit_button"
```

**Used by:** Execution logic for field mapping

### value_source: Optional[str]

**Purpose:** Where to get the value

**Valid values:**
- `"user"` - From user input
- `"profile"` - From user profile metadata
- `"static"` - Hard-coded value
- `"file_path"` - File path for uploads
- `"ai_generated"` - Generated by AI (questions)

**Used by:** ActionExecutor for value sourcing

### expected_value: Optional[str]

**Purpose:** Static value or expected result

**Examples:**
```python
expected_value="John"
expected_value="/path/to/resume.pdf"
expected_value="submit"
```

**Used by:** Validation and testing

### required: bool

**Purpose:** Whether step can be skipped

**Examples:**
```python
required=True   # Must execute
required=False  # Can skip if needed
```

**Used by:** Execution engine for optional steps

### metadata: dict[str, Any]

**Purpose:** Additional context

**Examples:**
```python
metadata={
    "platform": "linkedin",
    "form_type": "profile",
    "fields": ["firstName", "lastName", "headline"],
    "upload_allowed": True,
}
```

**Used by:** Advanced execution logic
